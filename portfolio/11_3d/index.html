Mapping 3D

Intervisibility between hillforts in South-west Shropshire

Tools I used

  • Made in QGIS
  • DTM data preparation done with GDAL at command line.

Data

Original hillfort kml download from Megalithic Portal search. Had to inspect data and move some hillfort points to highest point of hillfort according to the Lidar data. Is there a way of doing this automatically?

Height data is merged from 2 m lidar with OS Open data Terrain 50 to fill the gaps.

  • Defra Data Download
  • Terrain50 Opendata

What did I learn?

  • Using GDAL gdalbuildvrt to fill gaps in one raster with data from another.
  • When using Hillshade styling for DEM set Resampling > Zoomed in to Bilinear to remove line effect in close up. I should see what the possibilities are when generating actual hillshade layers too.
  • Tried Visibility Analysis plugin in QGIS, but was far too slow.
  • QGIS Line of Sight plugin worked well.
  • Using a layout template from the QGIS Layout-Hub

Was expecting there to be quite a lot of lines of sight between hillforts, though that was just a guess. In the event, there aren’t many. Maybe there are/were many more hillforts than in the list on Megalithic Portal?

What else would I like to do?

  • Work on the layout and the colours on the map! Not happy with it at the moment.
  • Now have book by Jo Jukes which does list a lot more hillforts. I’d like to try again with a more complete list.
  • I’m sure there should be more intervisibility than this, but how do I “ground truth” it?

Possible sources of more information

  • Jukes, J. 2021, Finding Home: Journeys with the Hillforts of Shropshire
  • Discovering Shropshire History
  • Historic England website

Jukes talks about forts being in clusters, or almost family groups (find page numbers?). Could I do some form of cluster analysis on the data?
